# Contributing

  Thank you for your interest in contributing.

  ## Workflow

  1. Fork the repository and create a topic branch from `main`.
  2. Make focused, well-scoped commits.
  3. Ensure all CI checks pass locally where possible.
  4. Open a Pull Request against `main` with a clear description of the change and its motivation.

  ## Code Review

  All changes land via Pull Request. Reviews focus on correctness, scope, and maintainability.

  ## Code of Conduct

  By participating, you agree to abide by the [Code of Conduct](./CODE_OF_CONDUCT.md).
