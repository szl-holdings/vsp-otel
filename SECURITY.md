# Security Policy

  ## Supported Versions

  The latest version on the default branch is the supported version. Older tags are not maintained.

  ## Reporting a Vulnerability

  Please report security vulnerabilities privately via [GitHub Security Advisories](https://github.com/szl-holdings/vsp-otel/security/advisories/new).

  - Do **not** open public issues for suspected vulnerabilities.
  - Include a clear description, reproduction steps, and impact assessment.
  - Maintainers aim to acknowledge reports within 5 business days.

  ## Disclosure

  Coordinated disclosure is preferred. We will work with reporters on a remediation timeline before any public disclosure.
